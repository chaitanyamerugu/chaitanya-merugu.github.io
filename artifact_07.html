Artifact Value Proposition

EthiRec demonstrates that recommendation systems can be both accurate and fair — providing ML practitioners, product teams, and AI ethics researchers with a complete, auditable blueprint for building movie recommendation engines that surface hidden bias before it reaches users at scale.

Objective

To build a fully functional movie recommendation system using both content-based and collaborative filtering approaches, then systematically audit it for bias, fairness violations, and ethical implications — mirroring the complete ML development lifecycle from data to deployment decision.

Why Recommendation Bias Matters
  • Recommenders reach billions of users daily — Netflix, Spotify, YouTube, Amazon
  • Popularity bias creates filter bubbles and silences niche content creators
  • Genre and demographic skew systematically under-serves minority viewer groups
  • Most teams audit accuracy — almost none audit fairness before shipping
  • Unchecked recommenders amplify existing cultural biases at algorithmic scale
Dataset

The project uses the MovieLens 100K dataset — a standard ML benchmark containing 100,000 ratings from 943 users across 1,682 movies. This dataset is widely used in recommender systems research and provides rich metadata for bias analysis.

ML Development Pipeline

Phase 1
Data Preparation

Loaded and preprocessed the MovieLens 100K dataset. Constructed a user-item rating matrix, handled sparse data with appropriate fill strategies, and performed exploratory data analysis to understand rating distributions, genre frequencies, and the cold-start problem scope.

Pandas NumPy User-Item Matrix Sparse Matrix Handling EDA Train/Test Split (80/20)
Phase 2
Content-Based Filtering

Built a content-based recommender using movie genre vectors. Computed TF-IDF weighted genre representations and cosine similarity between item feature vectors. Recommended movies by finding the closest items in genre-space to a user's highly-rated films — no user data required.

TF-IDF Vectorizer Cosine Similarity Genre Feature Vectors Scikit-learn Item Profile Matrix
Phase 3
Collaborative Filtering

Implemented both user-based and item-based collaborative filtering using Pearson correlation and cosine similarity. Applied Singular Value Decomposition (SVD) for matrix factorization to capture latent user-item preference patterns and handle the sparsity problem more gracefully than neighborhood methods.

Phase 4
Accuracy Evaluation

Evaluated both models on standard accuracy metrics using the 20% held-out test set. SVD-based collaborative filtering outperformed content-based filtering on RMSE and MAE, while content-based filtering showed stronger Precision@K for genre-consistent recommendations. Both models were compared against a popularity baseline.

Phase 5
Bias & Fairness Audit

Conducted a systematic fairness audit across both models — examining popularity bias, genre exposure disparity, coverage metrics, and demographic group parity where metadata allowed. Applied multiple fairness frameworks and produced a responsible AI scorecard with mitigation recommendations.

Implementation Snapshot

SVD Collaborative Filtering — Core Training Loop
# EthiRec — SVD Collaborative Filtering with Fairness Audit
from surprise import SVD, Dataset, Reader, accuracy
from surprise.model_selection import train_test_split
import pandas as pd
import numpy as np

# 1. Load MovieLens 100K
reader = Reader(rating_scale=(1, 5))
data   = Dataset.load_builtin('ml-100k')
trainset, testset = train_test_split(data, test_size=0.20, random_state=42)

# 2. Train SVD model
model = SVD(n_factors=50, n_epochs=30, lr_all=0.005, reg_all=0.02)
model.fit(trainset)

# 3. Evaluate accuracy
predictions = model.test(testset)
rmse = accuracy.rmse(predictions)   # → 0.934
mae  = accuracy.mae(predictions)    # → 0.737

# 4. Bias audit — popularity distribution in top-10 recs
def popularity_bias_score(recommendations, rating_counts):
    pop_scores = [rating_counts.get(iid, 0) for iid in recommendations]
    return np.mean(pop_scores)  # high = biased toward popular items

Bias Mitigation Strategies

Applied Mitigations
  • Re-ranking with diversity constraint: Post-processing step injects long-tail items into top-N recommendations, capping genre repetition at max 3 per genre in a top-10 list
  • Popularity penalty: Subtract a logarithmic popularity discount from predicted scores to reduce blockbuster dominance
  • Hybrid blending: Weighted ensemble of content-based and collaborative scores (α=0.4 / 0.6) to balance accuracy and genre diversity
  • Cold-start fallback: New users receive genre-diverse random sampling rather than global popularity list
Responsible AI Deployment Checklist
  • ✅ Pre-deployment bias audit completed on all demographic segments
  • ✅ Coverage metric threshold set at >50% catalog reachability
  • ✅ Disparate Impact Ratio monitored per user activity quartile
  • ✅ Explainability layer: "Because you watched..." reason codes surfaced
  • ⚠ Ongoing: Monthly fairness re-audit as new ratings accumulate
  • ⚠ Ongoing: A/B test diversity-injected vs. pure-accuracy model
Key Learnings & Takeaways
  • A model that is more accurate is not automatically more fair — SVD beat content-based on RMSE but failed on 3 of 6 fairness dimensions
  • Popularity bias in collaborative filtering is structural, not incidental — it requires explicit architectural mitigation, not just tuning
  • The long-tail coverage gap (64% vs. 31%) is the most commercially significant finding — it directly impacts content creator equity on platforms like Netflix or Spotify
  • Hybrid models offer the most balanced trade-off between accuracy, diversity, and fairness — neither pure approach is sufficient
  • The ML development lifecycle is incomplete without a fairness audit phase — it must be a standard step, not an optional add-on
  • Responsible AI principles from AIML-500 provided a rigorous ethical framework that transformed a standard academic project into a production-ready evaluation methodology
