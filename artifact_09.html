Introduction

Employee attrition is one of the most costly and disruptive challenges facing HR organizations worldwide. When an employee leaves, organizations face not just direct replacement costs — recruiting, onboarding, and training — but also indirect costs: lost institutional knowledge, disrupted team dynamics, and reduced productivity during transition periods. Industry research consistently estimates the total cost of replacing an employee at 50%–200% of their annual salary.

Traditional HR approaches to attrition are reactive: exit interviews happen after the decision is made, and retention conversations begin only after a resignation letter is submitted. Machine learning offers a fundamentally different paradigm — predictive, proactive, and personalized. By analyzing patterns in existing HR data, supervised learning models can identify at-risk employees weeks or months before they resign, enabling HR teams to intervene with targeted retention strategies.

This artifact documents the complete development of an employee attrition prediction pipeline using the IBM HR Analytics Employee Attrition dataset — from data exploration through model training, comparison, and evaluation — applying the supervised learning fundamentals covered throughout AIML-500. The project bridges my 5+ years of enterprise SAP HCM experience with the ML skills developed in this course, making the problem domain deeply familiar while the solution methodology is newly acquired.

Artifact Description

Objective

To build, train, and compare three supervised learning classifiers — Logistic Regression, Decision Tree, and Random Forest — on the IBM HR Analytics dataset, identify the best-performing model for predicting employee attrition, interpret its feature importances, and deliver actionable HR retention insights backed by model evidence.

Dataset — IBM HR Analytics
  • 1,470 employee records across 35 features
  • Target variable: Attrition (Yes/No — 16.1% positive class)
  • Key features: Age, JobRole, JobSatisfaction, OverTime, MonthlyIncome, YearsAtCompany, WorkLifeBalance, DistanceFromHome
  • Dataset is imbalanced — requires class-weight adjustment
  • No missing values — requires encoding for categorical variables

Step-by-Step Process

01
Exploratory Data Analysis (EDA)
Examined feature distributions, identified the class imbalance (83.9% No / 16.1% Yes), mapped correlations between features like OverTime and Attrition. Discovered that employees working OverTime were 3.1× more likely to leave. Used Pandas profiling and value_counts() to surface the most predictive categorical features before modeling.
02
Feature Engineering & Preprocessing
Converted binary categorical features (OverTime, Gender) via label encoding. Applied one-hot encoding to multi-class categoricals (Department, JobRole, MaritalStatus). Used StandardScaler for numerical features to normalize scale differences. Dropped non-predictive ID columns (EmployeeNumber, Over18, StandardHours). Final feature matrix: 46 columns after encoding.
03
Train/Test Split with Stratification
Applied an 80/20 stratified train-test split to preserve the class distribution in both sets. Stratification was critical given the 16.1% positive class rate — without it, the test set could accidentally contain too few attrition cases to evaluate recall meaningfully.
04
Model Training — Three Classifiers
Trained three supervised learning classifiers using scikit-learn, each with class_weight='balanced' to address the imbalanced dataset: (1) Logistic Regression with L2 regularization as a strong linear baseline; (2) Decision Tree Classifier with max_depth=5 to prevent overfitting; (3) Random Forest Classifier with 100 estimators as the ensemble method. All models were trained on the same features and split.
05
Evaluation & Model Comparison
Evaluated all three models on Accuracy, Precision, Recall, F1-Score, and ROC-AUC. For an attrition use case, Recall is the most critical metric — missing an at-risk employee (false negative) is far more costly than a false positive. Random Forest achieved the best overall performance with 87% accuracy and 0.81 ROC-AUC, while Logistic Regression produced the highest recall (79%) for the attrition class.
06
Feature Importance & HR Insight Generation
Extracted Random Forest feature importances to identify the top 10 attrition drivers. Translated these model outputs into HR-actionable recommendations — mapping each high-importance feature to a retention strategy. For example: OverTime as the #1 predictor maps directly to a workload management intervention; low MonthlyIncome combined with high YearsAtCompany signals unrecognized tenure, suggesting a compensation review process.

Top Attrition Drivers & HR Insights

Top 5 Feature Importances (Random Forest)
  • #1 — OverTime: Employees working overtime are 3.1× more likely to leave. Strongest single predictor.
  • #2 — MonthlyIncome: Low income relative to role level is a strong attrition signal, especially below $3,500/month.
  • #3 — Age: Younger employees (22–28) show the highest attrition rates; mid-career (35–45) are significantly more stable.
  • #4 — YearsAtCompany: Employees in their 1st–3rd year show peak attrition — the critical retention window.
  • #5 — JobSatisfaction: Low job satisfaction (score 1–2) correlates with 2.4× higher attrition probability.
Actionable HR Retention Recommendations
  • Overtime cap policy: Implement a monthly overtime hours threshold and trigger a manager alert when exceeded for 3+ consecutive weeks.
  • Early-tenure check-ins: Create structured 30/60/90-day retention conversations for all employees in Year 1–3.
  • Compensation benchmarking: Run quarterly salary benchmarking for roles where MonthlyIncome + YearsAtCompany signals divergence from market.
  • Satisfaction pulse surveys: Monthly micro-surveys (3 questions) to detect JobSatisfaction drops before they translate to resignation intent.
  • Predictive flagging: Deploy model output as a weekly at-risk score visible to HR business partners — not line managers, to prevent bias.

models = {

# Train each model and evaluate on held-out test set

Key Learnings from This Project
  • Class imbalance is the silent accuracy killer in HR datasets — a model that predicts "No attrition" for everyone achieves 83.9% accuracy but is completely useless. The class_weight='balanced' parameter was essential.
  • Random Forest outperforms Decision Tree on most metrics, but Logistic Regression won on Recall — a reminder that "best model" depends entirely on the metric that matters for your use case.
  • Feature importance from tree-based models is a bridge between ML and human decision-making — it translates opaque model weights into vocabulary that HR leaders can act on.
  • The most valuable output of this project was not the 87% accuracy number — it was the five HR interventions derived from feature importance. That's the difference between an ML experiment and an ML application.
