01 — Objective

Eliminate manual leave approval bottlenecks in Pennsylvania's statewide HR system by building a cloud-native BTP extension. The goal: reduce process turnaround time and improve compliance visibility — without touching the core S/4HANA system.

02 — Architecture

Side-by-Side Extension Pattern

Layer 01

CAP Backend (Node.js)

Core business logic implemented using SAP Cloud Application Programming Model. Defines CDS entities, service handlers, and custom approval orchestration rules.

CAP · CDS · Node.js
Layer 02

RAP-Based OData Services

ABAP RESTful Application Programming model exposes S/4HANA leave data as OData V4 services, consumed by the BTP extension layer.

RAP · OData V4 · ABAP
Layer 03

Fiori Elements UI

Annotation-driven, no-code frontend built on SAP Fiori Elements — delivering a consistent, accessible UI without custom frontend development.

Fiori Elements · SAP UI5
Layer 04

Build Process Automation

End-to-end approval routing configured in SAP Build Process Automation. Handles multi-level approvals, escalation paths, and SLA enforcement.

SBPA · Workflow · Forms

04 — Implementation Process

Build Sequence

1

Extension Architecture Design

Defined the side-by-side pattern using CAP (Node.js) as the extension layer, ensuring clean separation from the S/4HANA core. Mapped data flows and established BTP subaccount configuration.

2

RAP OData Service Design

Built RAP-based OData V4 services to expose leave request entities from S/4HANA. Consumed via SAP Business Accelerator Hub APIs to ensure up-to-date, stable integration contracts.

3

Fiori Elements UI Development

Configured an annotation-driven Fiori Elements application for leave request submission and status tracking. Delivered a responsive, accessibility-compliant UI without hand-written frontend code.

4

End-to-End Workflow Configuration

Configured approval routing, escalation paths, and SLA timers in SAP Build Process Automation. Implemented multi-level manager approval with automatic HR system write-back upon final decision.

5

S/4HANA Backend Integration

Integrated the extension with the S/4HANA backend via SAP Business Accelerator Hub APIs. Applied destination service and principal propagation to maintain secure, audited communication.
