SAP BTP Side-by-Side Extension:
Leave Approval Workflow

SAP BTPCAP (Node.js) RAPFiori Elements Build Process AutomationOData Services
Artifact Value Proposition

This extension shows government agencies how to modernize HR workflows without core modification — reducing leave approval turnaround by 30% and providing a replicable BTP extension pattern for enterprise HR use cases across public sector organizations.

Objective

To eliminate manual leave approval bottlenecks in Pennsylvania's statewide HR system by building a cloud-native SAP BTP side-by-side extension — reducing process turnaround and improving compliance visibility without modifying the core S/4HANA system.

Business Problem Solved
  • HR teams spending excessive time on manual leave routing via email/phone
  • No centralized audit trail for approval decisions
  • Core ECC modifications creating upgrade risk
  • Inconsistent policy enforcement across state agencies
Process — Step by Step
  1. Requirements Analysis: Mapped existing leave approval email chains and manual steps with HR stakeholders at IESO.
  2. Architecture Design: Selected SAP BTP side-by-side extension pattern to avoid core S/4HANA modification. Chose CAP (Node.js) for the service layer and RAP for OData exposure.
  3. CDS & OData: Extended standard CDS views using Custom Fields & Logic (CFL). Exposed leave request entities via OData V4 services.
  4. Fiori Elements UI: Built the leave request submission and manager approval UI using SAP Fiori Elements — list report and object page floor plans for zero custom UI code.
  5. Workflow Automation: Configured end-to-end approval routing using SAP Build Process Automation with conditional branching for leave types and org hierarchy.
  6. Integration: Connected to S/4HANA backend via SAP Business Accelerator Hub APIs for employee master and org structure data.
  7. Testing & Deployment: Validated across HR and Finance user groups in development and QA environments prior to production rollout.

Key Learnings & Takeaways
  • Side-by-side extensions are the safest modernization path for government systems with strict upgrade cycles
  • Fiori Elements dramatically reduces frontend development time when business objects align with standard floor plans
  • Build Process Automation's decision tables make policy rules transparent and maintainable by non-developers
  • API Hub integration decouples the extension from S/4HANA version changes
